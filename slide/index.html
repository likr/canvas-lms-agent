エージェントカスタマイズ

MCPの概要

  • オープン標準プロトコル: LLMがローカルまたはリモートのツールやデータにアクセスするための統一基準。
  • 動的クライアント連携: サーバーが提供するAPIエンドポイントとパラメータのスキーマ情報を動的に公開。
  • 安全性の担保: 認証鍵や接続情報をMCPサーバー側で隠蔽し、安全に実行環境に接続します。

Agent Skillsの概要

  • パッケージ化された指示: エージェントに必要な手順書（SKILL.md）、スクリプト、参照資料を格納。
  • 行動ガイドのインポート: ワークスペース検知により、関連するスキルが自動的にエージェントに供給されます。
  • 効率的なツール運用: ハマりやすい手順やルールを事前に教え込むことで、不要な試行錯誤をカット。

Canvas LMS API

Canvas LMSでは多種多様な操作を行うためのWeb APIが公式に公開されています。

アクセストークンをエージェントに認識させることで、コースの操作から採点管理まであらゆるオペレーションを自動化可能です。
  • トークンによる認可: 環境変数等でアクセストークンを設定し、安全にプログラムからCanvasを操作。
  • 実行可能操作: コースの一覧確認、課題情報の収集、ディスカッション投稿、学生提出物の採点やフィードバック書き込み。

技術構成

canvas-lms-mcp

  • 汎用ツール構成: search-canvas-api と call-canvas-api の2ツールに集約。
  • 1000超のAPI操作: エンドポイント一覧をAIに参照させ、無数の操作を動的かつ汎用的にカバーします。
  • メタデータの統合: 軽量でありながら網羅性の高いAPI呼び出し基盤を実現。

canvas-lms スキル

  • ノウハウの共有: エージェントが効果的にMCPを叩くためのガイドライン（SKILL.md）を内包。
  • ページネーション・役割対応: 複数ページのデータ取得や、教員/学生権限ごとのパラメータ分岐を詳細に指示。
  • 動的解決ルール: 静的IDのハードコードを排し、動的なID取得と連携を行う手順を確立。
